MSA Script Builder · the problem

The idea was never the hard part.

Kids can picture ambitious movies. What stops them isn't imagination — it's getting from a vague idea to a plan they can actually shoot, holding every step in your head at once. That's executive functioning, and for a kid with ADHD it's a wall. I know — I was that kid, at this academy. The Script Builder carries that load, so kids keep the part that's theirs: imagining, telling, performing.

03

The rules it runs on

The handoff — always

Claude ⇄ Staff ⇄ Kids

  • Staff type and read every prompt aloud.
  • Kids decide the story, start to finish.
  • Claude never sees a name or speaks to a child.

Non-negotiable safety

  • Kids never chat with the AI directly.
  • No names, photos, video, audio, or accounts.
  • Written for about a six-year-old reader.
  • No child's media is ever recorded or stored.

How it's built

  • One page, two modes — staff and kid.
  • A small serverless middleman holds the key.
  • It calls Claude and returns the reply.
  • Hosted on Vercel; state stays in the page.
